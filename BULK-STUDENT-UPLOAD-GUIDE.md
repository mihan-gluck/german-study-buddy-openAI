# Bulk Student Upload Feature - User Guide

## 📋 Overview

The Bulk Student Upload feature allows administrators to add multiple students to the portal at once using a CSV file. This feature uses the same fields as the Monday.com CRM integration, ensuring consistency across all student creation methods.

---

## 🎯 Key Features

- ✅ Upload multiple students at once via CSV file
- ✅ Uses same fields as Monday.com CRM (13 fields)
- ✅ Automatic RegNo and Password generation
- ✅ Optional welcome email sending
- ✅ Detailed validation and error reporting
- ✅ Download credentials for all created students
- ✅ Skip duplicate emails automatically
- ✅ Real-time progress tracking

---

## 🚀 How to Use

### Step 1: Access Bulk Upload

1. Log in as **Admin**
2. Navigate to **Admin Dashboard**
3. Click the **"Bulk Upload"** button in the action bar
4. The bulk upload panel will appear

### Step 2: Download CSV Template

1. Click **"Download CSV Template"** button
2. A file named `bulk_student_upload_template.csv` will be downloaded
3. Open the file in Excel, Google Sheets, or any spreadsheet application

### Step 3: Fill in Student Data

The CSV template contains the following columns:

#### Required Fields (Must be filled):
1. **name** - Student's full name
2. **email** - Unique email address (no duplicates)
3. **subscription** - Must be `SILVER` or `PLATINUM`
4. **level** - Must be `A1`, `A2`, `B1`, `B2`, `C1`, or `C2`
5. **studentStatus** - e.g., `ONGOING`, `COMPLETED`, `PAUSED`, `DROPPED`

#### Optional Fields (Can be left empty):
6. **medium** - e.g., `Online`, `Offline`, `Hybrid`
7. **batch** - e.g., `Batch A`, `Morning Batch`
8. **phoneNumber** - e.g., `+94771234567`
9. **address** - e.g., `Colombo, Sri Lanka`
10. **age** - e.g., `25`
11. **programEnrolled** - e.g., `German Language Course`
12. **leadSource** - e.g., `Facebook`, `Referral`, `Google Ads`

**Example CSV Content:**
```csv
name,email,subscription,level,studentStatus,medium,batch,phoneNumber,address,age,programEnrolled,leadSource
John Doe,john@example.com,PLATINUM,A1,ONGOING,Online,Batch A,+94771234567,"Colombo, Sri Lanka",25,German Language Course,Facebook
Jane Smith,jane@example.com,SILVER,A2,ONGOING,Offline,Batch B,+94771234568,"Kandy, Sri Lanka",30,Business German,Referral
```

### Step 4: Upload the CSV File

1. Click **"Select CSV File"** button
2. Choose your completed CSV file
3. The file name will appear next to the button
4. Choose whether to send welcome emails (checkbox)
5. Click **"Upload and Create Students"** button

### Step 5: Review Results

After upload completes, you'll see:

#### Summary Cards:
- **Total Rows** - Total students in CSV
- **Successful** - Students created successfully
- **Failed** - Students that couldn't be created (with reasons)
- **Skipped** - Duplicate emails (already exist)

#### Detailed Tables:
- **Successfully Created** - Shows name, email, RegNo, password, email status
- **Failed** - Shows row number, data, and failure reason
- **Skipped** - Shows row number, data, reason, and existing RegNo

### Step 6: Download Credentials

1. Click **"Download Credentials"** button
2. A CSV file with all credentials will be downloaded
3. File name format: `student_credentials_YYYY-MM-DD.csv`
4. Contains: Name, Email, RegNo, Password, Email Sent status

## 📊 Field Validation Rules

### Name
- **Required:** Yes
- **Format:** Any text
- **Example:** `John Doe`

### Email
- **Required:** Yes
- **Format:** Valid email format
- **Unique:** Must not already exist in system
- **Example:** `john@example.com`

### Subscription
- **Required:** Yes
- **Valid Values:** `SILVER`, `PLATINUM`
- **Case:** Case-insensitive (will be converted to uppercase)
- **Example:** `PLATINUM`

### Level
- **Required:** Yes
- **Valid Values:** `A1`, `A2`, `B1`, `B2`, `C1`, `C2`
- **Case:** Case-insensitive (will be converted to uppercase)
- **Example:** `A1`

### Student Status
- **Required:** Yes
- **Format:** Any text (typically: ONGOING, COMPLETED, PAUSED, DROPPED)
- **Example:** `ONGOING`

### Medium
- **Required:** No
- **Format:** Any text
- **Example:** `Online`, `Offline`, `Hybrid`

### Batch
- **Required:** No
- **Format:** Any text
- **Example:** `Batch A`, `Morning Batch`

### Phone Number
- **Required:** No
- **Format:** Any text (no validation)
- **Example:** `+94771234567`

### Address
- **Required:** No
- **Format:** Any text (can include commas - use quotes)
- **Example:** `"Colombo, Sri Lanka"`

### Age
- **Required:** No
- **Format:** Number
- **Example:** `25`

### Program Enrolled
- **Required:** No
- **Format:** Any text
- **Example:** `German Language Course`

### Lead Source
- **Required:** No
- **Format:** Any text
- **Example:** `Facebook`, `Referral`, `Google Ads`

## 🔐 Auto-Generated Fields

These fields are automatically generated by the system:

### Registration Number (RegNo)
- **Format:** `STUDXXX` (e.g., `STUD042`)
- **Generation:** Sequential 3-digit number
- **Unique:** Yes

### Password
- **Format:** `Student{last3digits}@{year}`
- **Example:** `Student042@2026` (for RegNo STUD042 in 2026)
- **Sent to:** Student via email (if enabled)

### Role
- **Value:** Always `STUDENT`
- **Cannot be changed:** Yes

### Created At
- **Value:** Current timestamp
- **Format:** ISO 8601

### Last Credentials Sent
- **Value:** Email sent timestamp (if email was sent)
- **Format:** ISO 8601

## ⚠️ Common Errors and Solutions

### Error: "Email is required"
**Cause:** Email field is empty  
**Solution:** Fill in the email field for that row

### Error: "Subscription must be SILVER or PLATINUM"
**Cause:** Invalid subscription value  
**Solution:** Use only `SILVER` or `PLATINUM` (case-insensitive)

### Error: "Level must be A1, A2, B1, B2, C1, or C2"
**Cause:** Invalid level value  
**Solution:** Use only valid CEFR levels

### Error: "Student Status is required"
**Cause:** Student Status field is empty  
**Solution:** Fill in the student status (e.g., ONGOING)

### Skipped: "Email already exists"
**Cause:** Email is already registered in the system  
**Solution:** This is normal - student already exists. Check existing RegNo in results.

### Error: "Failed to send email"
**Cause:** Email service error  
**Solution:** Student was created but email failed. Use "Resend Credentials" button in Admin Dashboard.

## 💡 Best Practices

### 1. Prepare Your Data
- Clean your data in Excel/Sheets before uploading
- Remove any duplicate emails
- Verify all required fields are filled
- Use consistent formatting

### 2. Test with Small Batch First
- Upload 2-3 students first to test
- Verify results before uploading large batches
- Check email delivery

### 3. Handle Addresses with Commas
- If address contains commas, wrap in quotes
- Example: `"123 Main St, Colombo, Sri Lanka"`

### 4. Email Sending
- For large batches (100+), consider disabling email sending
- Download credentials and send manually
- Prevents email service rate limiting

### 5. Keep Credentials Safe
- Download credentials CSV immediately
- Store securely
- Don't share publicly

### 6. Review Failed/Skipped
- Check failed rows for data issues
- Fix and re-upload failed rows
- Skipped rows (duplicates) are normal

## 🛠️ Technical Details

### API Endpoint
```
POST /auth/bulk-upload-students
```

### Request Body
```json
{
  "students": [
    {
      "name": "John Doe",
      "email": "john@example.com",
      "subscription": "PLATINUM",
      "level": "A1",
      "studentStatus": "ONGOING",
      "medium": "Online",
      "batch": "Batch A",
      "phoneNumber": "+94771234567",
      "address": "Colombo, Sri Lanka",
      "age": "25",
      "programEnrolled": "German Language Course",
      "leadSource": "Facebook"
    }
  ],
  "sendEmails": true
}
```

### Response
```json
{
  "success": true,
  "message": "Bulk upload completed",
  "summary": {
    "total": 10,
    "successful": 8,
    "failed": 1,
    "skipped": 1
  },
  "results": {
    "successful": [...],
    "failed": [...],
    "skipped": [...]
  }
}
```

### Authentication
- Requires: Admin role
- Token: Bearer token in Authorization header

## 🔍 Troubleshooting

### CSV File Not Accepted
- Ensure file extension is `.csv`
- Check file is not corrupted
- Try re-saving from Excel as CSV

### All Rows Failed
- Check CSV format is correct
- Verify headers match template exactly
- Check for special characters

### Emails Not Sending
- Verify email configuration in `.env`
- Check email service is running
- Review backend logs for errors

### Slow Upload
- Large files (1000+ rows) may take time
- Progress bar shows activity
- Don't refresh page during upload
